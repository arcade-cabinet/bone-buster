# Objexoom easter egg

A DOOM-flavored easter egg triggered by `?objexoom`. Sibling to the
`?fuzzyfuzzycutecute` llama overlay — same lifecycle contract, same mount
point, same query-string-as-opt-out behavior.

## Inspirations

- **Logic source:** [carlini/js13k2019-yet-another-doom-clone](https://github.com/carlini/js13k2019-yet-another-doom-clone).
  We port the *shape* of its gameplay — random sector-walled corridors,
  hitscan weapon, billboarded enemies, locked-door progression — not its WebGL
  pipeline. Objexoom uses react-three-fiber.
- **Visuals:** PSX-era retro 3D, sourced from `/Volumes/home/assets/3DPSX/`.
- **Tokens:** Objexiv MUI theme — indigo `#6172f3` → violet `#a855f7` → amber
  `#f59e0b`, ink `#060912`. HUD respects light/dark.

## Trigger

| Param          | Value         |
| -------------- | ------------- |
| Query string   | `?objexoom`   |
| Mount          | `src/components/client-providers.tsx` alongside `LlamaEasterEgg` |
| Forward param  | Yes — clicking links + history pushState carries `?objexoom` forward, matching the llama contract |
| Opt-out        | Remove the query string |
| Concurrent llama | Both can coexist; objexoom canvas is `pointer-events: auto` only while pointer-lock is engaged |

## High-level architecture

```
src/client/lib/objexoom-easter-egg.ts          — query-string contract
src/client/components/ObjexoomEasterEgg.tsx    — gating wrapper, mirrors LlamaEasterEgg shape
src/client/components/LazyObjexoom.tsx         — Suspense + dynamic import
src/client/games/objexoom/
  engine.ts                                    — map gen, AI, weapon, collision (pure TS)
  ObjexoomScene.tsx                            — r3f <Canvas> + scene graph
  ObjexoomHUD.tsx                              — health, ammo, kill-count, "ESC to leave" hint
  PlayerController.tsx                         — pointer-lock + WASD + fire
  assets.ts                                    — manifest of GLB paths
  constants.ts                                 — tile size, palette, balance numbers
public/assets/games/objexoom/                  — chosen GLBs copied from 3DPSX
```

## Engine — what we port from the reference

| Reference concept | Objexoom port |
| --- | --- |
| Compressed turtle-graphics maps | **Replaced.** Procedural grid map (12×12 cells, seeded PRNG, drunk-walk corridor carver + locked-door room). One random map per session — no level progression. |
| Sector-based polygon walls with floor/ceiling heights | **Replaced.** Single-height grid: each cell is empty, wall, door, or spawn. Walls = `wall_hr_1.glb` instances at cell centers. Floor = tiled `floor_ceiling_hr_1.glb`. |
| Sprite enemies with billboarded textures | **Ported.** Skeleton warriors (`PSX Dungeon Skeleton Warrior/Skeleton_warrior.glb`) — animated GLBs rendered as actual 3D meshes, not billboards (we have the budget). |
| Hitscan weapon + projectile | **Ported.** Hitscan only — pistol-style. Raycast from camera forward, deal damage at first hit. |
| Enemy AI: sees player → walks → attacks | **Ported.** Line-of-sight check via grid cell raster, A* on the grid (`engine.ts`) to chase, melee on adjacent cell. |
| Locked door + key | **Ported, minimal.** One locked door blocks an alcove with a treasure (`PSX Dungeon Loot Pack/Treasure.glb`) and the exit portal. Key = floor-pickup, single instance. |
| Game over / win | **Ported.** Death → "RIP. ESC to return." HUD overlay. Win (touch portal) → "OBJEXIV OVERDRIVE. ESC to return." |

We do NOT port: dynamic shadow lighting, custom shaders, audio compression,
multi-level state, JSFXR audio (we use a single hard-coded WebAudio click for
the weapon firing — no sprawling audio subsystem).

## Determinism

`engine.ts` exports a pure `generateMap(seed: number): ObjexoomMap`. The seed
is `Date.now()` per fresh trigger, but a single map persists for the session
once generated — leaving and re-entering the same URL keeps the same map until
full page reload. This makes the map testable: pass a fixed seed in tests,
assert structural invariants (player spawn reachable from exit, exit reachable
only via locked door, ≥3 enemy spawns).

## HUD (Objexiv brand)

- Top-left: health bar — gradient `linear-gradient(135deg, #6172f3, #a855f7)`, ink fill at low health turns amber `#f59e0b`.
- Top-right: kill count + "PORTAL" indicator (pulses violet when key collected).
- Bottom center: crosshair, indigo dot with violet ring.
- Bottom-left: "ESC to leave OBJEXOOM" hint, fades in after 2s.
- Font: Poppins for the title overlay, Inter elsewhere — matches `src/lib/theme.ts`.

## Controls

- **Click canvas** → request pointer lock; entering pointer lock starts the game loop.
- **WASD** = move on grid (with collision against wall hitboxes — circle-vs-AABB).
- **Mouse** = look.
- **Left click** = fire pistol.
- **ESC** = release pointer lock; canvas pauses, "ESC to leave OBJEXOOM" remains.

## Coexistence with the llama overlay

When both `?objexoom` AND `?fuzzyfuzzycutecute` are present, both render.
Llama overlay is non-blocking and stays on the DOM layer above the Objexoom
canvas — surprise discovery doubles as bug.

## Test surface

1. **Unit** — `objexoom-easter-egg.test.ts`: query param detection + URL
   forwarding (mirror `llama-easter-egg.test.ts`).
2. **Unit** — `engine.test.ts`: `generateMap(SEED)` produces map where (a)
   player spawn is reachable, (b) treasure room is gated behind the locked
   door, (c) ≥3 enemy spawns exist, (d) re-running with same seed = same
   output.
3. **Browser** — `ObjexoomEasterEgg.test.tsx`: with `?objexoom`, canvas mounts,
   HUD renders, ESC unmounts; 9-point coverage per `docs/BROWSER_TESTING_STANDARD.md`.
4. **Visual** — screenshot the canvas in headless mode and verify it isn't
   pitch black (the existing pattern from `LlamaEasterEgg` browser tests).

## Out of scope (for THIS branch)

- Multi-level progression
- Audio beyond a single pistol-fire WebAudio blip
- Mobile touch controls (desktop-only easter egg; mobile sees an "open on desktop" hint)
- Online leaderboards or save state
- A second weapon

## What shipped (2026-05-13 update)

The branch that closed this spec went considerably beyond the original
"first playable slice". Tracking the over-delivery here so the spec
doesn't lie about the runtime:

**Section A — sector-map runtime**
- Discriminated `ObjexoomMap` union (`grid` vs. `sectors`) with
  dispatchers (`resolveCollisionAny`, `hasLineOfSightAny`,
  `castRayAny`) so the rest of the runtime is map-shape-agnostic.
- Polygonal sector collision via closest-point-on-segment + circle
  pushout, portal-edge detection by shared-vertex matching with
  matching floor heights.
- Turtle-graphics decoder + `loadRefLevel(0..4)` bridges the js13k
  reference's encoded level stream into runtime `ObjexoomSectorMap`s.
- `<SectorMapGeometry>` r3f component renders sector polygons as
  floor / ceiling shapes + per-edge wall quads.

**Section AA — headed e2e harness**
- `?objexoomDebug` exposes `window.__objexoom.{getState, start,
  teleport, fire, killAllEnemies, collectKey, triggerWin}`.
- `tests/e2e/objexoom.spec.ts` drives 7 scenarios in real headed
  Chromium via `pnpm test:e2e:objexoom:headed`.

**Section B — multi-level runs**
- `GameStatus` gains `"transitioning"`. Pure `runStatsReducer` +
  `advanceLevel()` track `runStartAt / runLevelsCleared /
  runTotalKills / runTotalDamageTaken`. After 5 clears, MISSION
  COMPLETE shows the run stats. Procedural mode chains by
  re-rolling the seed; ref mode chains by advancing
  `settings.level`.

**Section C — reference enemy AI**
- `EnemyKind` gains `"imp"`. FSM port in `enemyAi.ts` mirrors
  `BaseEnemy._fsm_advance` (states 0=patrol, 1=chase, 3=shoot) with
  `gethelp()` propagation. `EnemyBullet` actor (`stepEnemyBullet`)
  handles wall/player/expired branches. Imps explode-on-death into
  amber particle bursts.

**Section D — pickup parity**
- Health pickup re-tinted amber. Chaingun ammo → indigo cell box.
  Shotgun ammo → amber shell pair with brass primer caps.
- Decorative treasure chest stamps every exit.
- `ParticleBurstField` listens for `objexoom:burst` events,
  renders 4-12 motes per burst with gravity + 350ms fade.

**Section E — polish**
- Tip carousel cycles 4 DOOM-flavored hints on the landing.
- Global ESC on landing exits the egg.
- "RESUME RUN" surfaces when MAIN MENU is hit mid-run with a
  preserved seed.
- `useReducedMotion` gates landing tip rotation.
- `<noscript>` graceful fallback documented in
  `ObjexoomEasterEgg.tsx`.

**Test surface** (final): 95 unit tests, 5 browser tests, 7 e2e
tests. All green at branch tip.

**No longer "out of scope"** (the original spec called these out):
multi-level progression, mobile touch controls — both shipped. The
"three uses of the character factory" architectural question got an
answer: the union/dispatcher approach lets the same Scene render
both procedural grids and ref-decoded sector polygons without a
second runtime.
